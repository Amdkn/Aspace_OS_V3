## 🧠 Concepts Clés à Haute Densité Sémantique
- **<Chatbot vs Assistant de Code vs Agent Autonome (Triangle d'Or IA 2026)>** : Yassine Sdiri pose la taxonomie canonique : (1) **Chatbots** (Claude.ai, ChatGPT, Gemini, Mistral) = génératifs texte, répondent à un prompt, **nécessitent validation humaine à chaque étape**, pas d'action autonome. (2) **Assistants de code** (Claude Code, Codex) = dans le terminal/IDE, codent avec vous, adaptés à un projet en cours mais **nécessitent votre présence**. (3) **Agents autonomes** (Hermes, OpenClaw) = vivent sur serveur, exécutent des tâches récurrentes **sans supervision**, notifient via Telegram/WhatsApp en fin de tâche. Pour Tilly, c'est la carte de routing : à chaque moment, elle choisit l'outil aligné avec son intention (réponse ponctuelle = chatbot, code en cours = assistant, tâche récurrente = agent autonome).
- **<OpenClaw vs Hermes : Stabilité et Pré-configuration>** : OpenClaw (racheté par OpenAI pour >1 Md$, sorti fin décembre 2024) a ouvert la voie mais souffre de bugs à chaque update (configurations qui cassent, réinstallations forcées). Hermes (sorti quelques mois après) a validé les fondations pour que **tout le monde puisse l'utiliser** sans profil technique. Pré-configuration : mémoire déjà réfléchie pour fonctionner sur toutes les conversations, **>50 skills inclus dès l'installation** (vs Claude Code où il faut les créer/télécharger un par un), multicanal (WhatsApp, Discord, Slack, Telegram), boucle d'auto-amélioration intégrée. Pour Tilly, c'est l'archétype du **"AI for the rest of us"** : pas besoin d'être dev, l'agent marche out-of-the-box.
- **<Loop d'Auto-Amélioration par Skill Auto-Créé>** : Quand Hermes résout une tâche nouvelle via un schéma de réflexion, **il crée automatiquement un skill pour ne plus jamais avoir à résoudre cela**. Plus Tilly utilise Hermes, plus il devient efficace, plus il consomme moins de tokens (parce qu'il sait tout de suite quoi faire). C'est l'application directe du pattern Phase 2 de l'OpenClaw Heartbeat : auto-amélioration sans A0 GO. Pour Tilly, c'est l'**économie de tokens par capitalisation** : le premier run coûte cher, les suivants sont gratuits.
- **<Multicanal + Multi-Provider + Multi-Modèle = Souveraineté de Couplage>** : Hermes se branche sur **n'importe quel modèle** (Deepseek, Alibaba Qwen, Mistral, Llama, Claude, OpenAI) et **n'importe quel canal** (Telegram, WhatsApp, Discord, Slack). L'argument souverain : si Tilly installe un modèle open source sur sa machine, les données restent chez elle (vs Cloud Anthropic/OpenAI). C'est l'application directe de la doctrine A'Space Test Key Pragma : pas de vendor lock-in, capacité à switcher en 1 commande. Le pattern est : `Hermes model` → choisir provider → login → c'est fait.
- **<Workspace Kanban + Multi-Agents (Mode "Écurie de Squads")>** : L'interface Hermes inclut une vue Kanban (comme Notion) où Tilly peut empiler des tâches que l'agent va traiter en continu (avec différents sous-agents, différents modèles, différents métiers) — "tant qu'il est autonome et peut travailler nonstop, on n'a pas envie qu'il arrête de travailler moyennant les consommations de token". Pour Tilly, c'est l'application de la doctrine A'Space L2 Business Pulse : **chaque squad = un agent spécialisé**, on empile les missions et l'écosystème les distribue.
- **<Coût ÷10 vs Cloud Code + Choix du Modèle par Tâche>** : "Il va vous coûter 10 fois moins cher" — le framework n'est pas une IA en tant que telle (Hermes a aussi son modèle interne), c'est une plateforme qui **utilise** d'autres modèles. Conséquence pratique : pour la création de posts LinkedIn récurrents, Deepseek coûte infiniment moins cher qu'Opus 4.7 sans perte de qualité. Pour l'agent CEO qui délègue les tâches entre sous-agents, Opus 4.8 est justifié. Pour Tilly, c'est la **stratégie d'orchestration par coût** : 1 agent CEO sur Opus 4.8 (stratégie), 10 agents squads sur Deepseek/Claude Sonnet (exécution).
- **<Hermes vs Claude Code : Complémentarité, Pas Concurrence>** : Yassine tranche le débat : "chacun a son terrain et ensemble il couvre tout". Claude Code = desktop, supervision, projets d'envergure. Hermes = serveur, autonomie, tâches longues. Tilly peut **piloter Claude Code depuis Hermes** (et vice-versa) — c'est l'orchestrateur ultime. Le bon usage : "Claude Code pour mes projets d'envergure et pour gérer toutes mes entreprises, et Hermes en parallèle sur des tâches beaucoup plus longues qui va travailler de manière autonome".
