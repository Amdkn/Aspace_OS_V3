# Wishlist: Agentic Local Adapters (ALA) Standard [GOD MODE]
## Target Layer: L0 / L2 (Bedrock & Business Pulse)
## Purpose: Transformer n'importe quel logiciel local ou open-source en une capacité native de l'OS via le standard "ALA". S'affranchir des APIs Cloud et du MCP pour un contrôle souverain total et un "God Mode" agentique.
## Strategic Alignment: Méta-Science / Sovereign Infra / Antifragilité H1-H3

### Required Components
- [ ] **L'ALA Registry (Strate L0)**
  - *Rationale* : Un registre centralisé sur le Bedrock listant tous les logiciels "ingérés" et leurs schémas de commande (JSON). Ce registre est la source de vérité pour tous les agents.
- [ ] **Le Pipeline d'Ingestion Automatisé (The Ingestor)**
  - *Rationale* : Un workflow (basé sur CLI Anything) capable d'analyser le code source d'un outil (ex: Blender, Excalidraw) et de générer l'adaptateur ALA correspondant avec documentation agentique.
- [ ] **Intégration "Reflexion-First" (SDD to DDD)**
  - *Rationale* : Injecter dans le système de pensée des agents (A0 à A3) la capacité de considérer l'ingestion d'un logiciel existant comme une alternative viable au développement "from scratch".
- [ ] **Bridge H1 (Open Claw) & H3 (ADK)**
  - *Rationale* : Mapping des schémas ALA pour qu'ils soient consommables nativement par les modèles de raisonnement Google ADK et les agents d'exécution Open Claw.

### Explicitly Excluded (Anti-Wishlist)
- [ ] **Dépendances Runtime Externes**
  - *Pourquoi* : Un ALA doit pouvoir tourner offline. Aucune dépendance à un serveur tiers ou à une plateforme "as a service" n'est tolérée dans le standard ALA.
- [ ] **Automation Visuelle (Mouse/Keyboard simulation)**
  - *Pourquoi* : On ne veut que du contrôle par commandes et modèles d'états. Pas de clics, pas de fragilité.

### Non-Negotiable Constraints
- **Security**: Isolation stricte via Sandbox pendant la phase d'ingestion (analyse de code). Les permissions d'un ALA sont limitées au domaine IndexDB demandeur.
- **Performance**: Ingestion asynchrone (10-15 min) avec notification de "Compétence Acquise" dans l'OS Shell.
- **Dependencies**: Nécessite une instance locale de Gemini CLI ou OpenClaw pour piloter le registry.

### TVR Assessment
- **Teachable**: ✅ (T) L'ALA produit sa propre doc agentique. L'agent "apprend" à utiliser le logiciel en quelques secondes.
- **Valuable**: ✅ (V) Accès illimité à l'écosystème software sans surcoût API.
- **Repeatable**: ✅ (R) Processus d'ingestion 100% scriptable et idempotent.
