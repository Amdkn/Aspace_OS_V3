## 4. Debt list (classified per Picard workflow)

### Critical (blocks production)
- **No git repo** ‚Äî `git status` returns `fatal: not a git repository`. Required for Phase 6 GitHub push.
- **No `package.json`** ‚Äî no `engines.node`, no `npm install` reproducible path.
- **No build / no TypeScript** ‚Äî script `useState`/`setState` inside HTML `<script>` block (lines 440-620). Vanilla `React.createElement` without JSX. Zero type safety.
- **CDN font loading** ‚Äî `<link href="https://fonts.googleapis.com/css2?...">` causes FCP delay on first paint. Migration target: `next/font/local`.
- **Window-bound React** ‚Äî `support.js` does `window.React` global binding (legacy pattern, anti-ESM). Migration: standard ESM `import { useState, useEffect } from 'react'`.

### High (degrades DX + SEO)
- **Inline styles everywhere** ‚Äî `style="..."` on every node (~400+ occurrences). No CSS modules, no Tailwind. Migration: Tailwind utility classes + design tokens in `src/design/tokens.ts`.
- **Claude Design custom templating** ‚Äî `<sc-for list="..." as="..." hint-placeholder-count="...">` requires runtime support (`support.js`). Migration: native React `.map()`.
- **No SEO / metadata** ‚Äî zero `<title>`, `<meta name="description">`, OpenGraph tags, favicon config. Migration: Next.js `metadata` export + `app/layout.tsx`.
- **Claude Design runtime is non-freezable** ‚Äî `support.js` is auto-generated by `dc-runtime` build (header comment: "do not edit. Rebuild with `cd dc-runtime && bun run build`"). Coupling to a tool we don't control = maintainability risk.

### Medium (nice-to-have)
- **No favicon** ‚Äî `.thumbnail` exists but is not linked.
- **No `robots.txt` / `sitemap.xml`** ‚Äî SEO basics missing.
- **No accessibility audit** ‚Äî color contrast `#F5EFE0` on `#0A0E16` is excellent (AAA), but ARIA landmarks could be more semantic (`<main>`, `<nav>` role).
- **No tests** ‚Äî zero coverage (per common/testing 80% min).
