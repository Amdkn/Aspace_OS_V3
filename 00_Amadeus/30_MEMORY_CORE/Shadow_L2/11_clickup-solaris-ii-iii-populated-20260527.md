## Pattern de nommage canonique tasks (à venir)

Convention universelle à appliquer dès la première task créée :

```
[SOP-L2-{DOMAIN}-{NN}] {CLIENT_ID} — {action courte}
```

Exemples :
- `[SOP-L2-SALES-001] CL-acme-corp — Qualifier lead inbound depuis Apollo`
- `[SOP-L2-FINANCE-001] CL-acme-corp — Émettre invoice Stripe février`

Le tag `SOP-L2-XXX-NN` permet le lookup croisé vers Notion MASTER_SOP_DB sans dépendre d'un Custom Field (économise budget ClickApps du plan Free).

## Custom Field doctrine (anti-budget-burn)

⚠️ **Décision** : NE PAS créer de Custom Field `SOP_ID` workspace-wide. Raison :

- Plan Free a un quota strict d'usage Custom Fields ("ClickApps uses")
- Le pattern `[SOP-L2-XXX-NN]` dans le **titre** des tasks remplace fonctionnellement le Custom Field
- Recherche full-text ClickUp matche déjà les préfixes en crochets
- Pour cross-référencement Notion → ClickUp, on utilisera **Symphony worker** (Phase 2) qui matchera par regex sur titre

À ré-évaluer **uniquement** si upgrade vers ClickUp Business+.
