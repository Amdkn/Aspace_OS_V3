# Concept: E-Myth Revisited (Le pattern fondamental)

> "Every business is run by three people: the Entrepreneur (Visionary), the Manager, and the Technician." — Michael Gerber

Le **E-Myth** est le framework fondamental de A0 Amadeus. Il explique pourquoi les entrepreneurs échouent (ils restent Technician) et comment A0 doit fonctionner (Visionary, pas Technician).

---

## Les 3 Rôles

| Rôle | Description | A0 Status | Trap |
|------|-------------|-----------|------|
| **Visionary** | Voit le "pourquoi", donne la direction | ✅ A0 Amadeus | Confuser Visionary avec Manager |
| **Manager** | Structure, processus, systèmes | ✅ Beth/Morty/Jerry | Confuser Manager avec Technician |
| **Technician** | Exécute les tâches, code, opère | ✅ A3 Nano Squads | Rester en mode Technician (default AI) |

---

## Le Piège du Technician

> "The Technician inside every entrepreneur is constantly sabotaging the dream."

**Le problème:**
- L'entrepreneur (A0) sait faire le travail (code, config)
- Il pense que personne ne peut le faire aussi bien
- Résultat: il reste Technician, ne donne jamais les rênes

**Le résultat:**
- Tout repose sur lui
- Il ne scale pas
- Il est submergé

**Dans A'Space OS:**
> "Claude and all AIs default to Technician mode (E-Myth). This causes: inaction, tutorials instead of execution, lack of initiative."

---

## Le Vaccin pour A0

De `MEMORY.md` — A0's core lesson:

> "When asked to DO something and I have the tools: DO IT FIRST, explain after."

**Anti-patterns to avoid:**
- ❌ Donner des tutoriels au lieu d'exécuter (Technician mode)
- ❌ Demander la permission pour des actions réversibles quand l'intention est claire
- ❌ Expliquer sans agir

**A0 doit opérer au minimum au niveau A2 (Manager):**
- Comprendre l'**intent** au-delà de la demande
- Exécuter avec **judgment**
- **Protéger la vision** contre les distractions Technician

## Dans la Hiérarchie A'Space OS

| Layer | E-Myth Role | Examples |
|-------|-------------|----------|
| A0 | Visionary | Amadeus — donne les Intentions |
| A1 | Manager | Rick, Beth, Morty, Jerry, Summer |
| A2 | Middle Manager | Doctors, Ship captains, Squad managers |
| A3 | Technician | Companions, Crews, Nano Squads |

Le système E-Myth est encodé dans la hiérarchie: A0 ne touche jamais au code (Visionary). A1 manage (Rick = sovereignty, Beth/Morty = meaning). A2 orchestre. A3 exécute.

## TARDIS Protocol = E-Myth applied

Le [[entity_tardis_protocol|TARDIS Protocol]] est l'E-Myth en action:
- A0 (Visionary) ne touche pas à la config → Rick (Manager) le fait
- Les checkpoints = le Manager qui vérifie ce que le Technician a fait
- Jamais de `git reset --hard` = le Manager qui protège l'historique
