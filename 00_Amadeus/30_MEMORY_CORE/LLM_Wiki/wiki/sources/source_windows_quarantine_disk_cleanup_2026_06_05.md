# Windows Quarantine Disk Cleanup - 2026-06-05

## Executive Summary

The PS4 external Seagate 4 TB disk was repurposed as a Windows quarantine disk:
`Q:\` / `ASPACE_QUARANTINE_4T` / NTFS.

The cleanup policy is copy first, manifest, then purge only after the `Q:` copy exists.

## Evidence

- Quarantine root: `Q:\_QUARANTINE_INBOX\bulk-quarantine-20260605-001738\`
- Copy manifest: `Q:\_QUARANTINE_MANIFESTS\bulk-quarantine-20260605-001738.csv`
- Purge manifests: `Q:\_READY_FOR_PURGE_AFTER_REVIEW\`
- Post-purge audit: `C:\Users\amado\storage-audit\storage-audit-after-purge-summary.md`
- WSL docs evidence: Context7 `/microsoftdocs/wsl`, commands `wsl --export` and `wsl --unregister`.

## Preserved Decisions

- `C:\Users\amado\Downloads` folder is preserved, but its contents were purged after quarantine.
- `C:\Archives_PreCowork`, `C:\Users\amado\Archives`, `C:\HPIADownloads`,
  `C:\Users\amado\ASpace_OS_V2\_TRASH`, and `C:\rtr98D1.tmp` were purged from `C:`
  only after their `Q:` copies were verified.
- `C:\Windows\Installer` is not manually deleted.
- WSL optimization must export a distro before unregistering it.
- `ASpace-L0` WSL was exported to `Q:\_QUARANTINE_INBOX\wsl_exports\ASpace-L0_20260605.vhdx`
  before `wsl --unregister ASpace-L0`.

## What Is Obsolete

- PS4 partitioning on the Seagate drive.
- Root temp file `C:\rtr98D1.tmp`.
- Local copies of quarantined archive/download blocks after successful copy.

## Archive Decision

The Windows cleanup is an active quarantine system, not a final archive. `Q:` is the
reviewable holding area before irreversible long-term deletion.

## Result

- `C:` moved from roughly 67 GiB free to 479.40 GiB free.
- `Q:` stores the quarantined archive/download blocks and the WSL export.
- The 50% free-space threshold on the Windows SSD was reached.

## Residual Risk

- WSL distro removal is destructive after `wsl --unregister`; export must complete first.
- Some application caches or model stores may be regenerated by their apps after deletion.
- Windows system folders such as `WinSxS` and `Installer` require official cleanup tools,
  not manual deletion.
