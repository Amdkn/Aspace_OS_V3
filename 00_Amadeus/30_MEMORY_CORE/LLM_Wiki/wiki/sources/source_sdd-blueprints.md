## Thèmes majeurs

### 1. Autonomie Agentique (SDD-001)

> Le Solarpunk Kernel doit être antifragile: il ne cherche pas à éviter les failures mais à se renforcer à travers eux.

L0.3 introduit le concept d'**autonomie agentique** — chaque agent L2/L3 opère avec une souveraineté locale, mais toujours dans le cadre du TARDIS Protocol. Pas d'override possible sans ADRs.

### 2. TARDIS Protocol (SDD-003)

Le protocole TARDIS est le **version control pour agents**. Chaque agent a:
- Un historique (pas d'écrasement)
- Des checkpoints (à chaque ADR nouveau)
- Une traçabilité (qui a changé quoi, quand)

Le TARDIS Protocol est le lien entre [[concept_sovereignty]] et l'exécution technique.

### 3. Initiative ALPHA (SDD-004)

Quand Rick est absent, le système doit continuer. **Initiative ALPHA** = les Nano Squads peuvent exécuter des tâches de niveau 1-day sans approbation préalable si:
1. La tâche correspond à un ADR existant
2. Le guardian A2 est joignable (pas DLQ)

### 4. Jumeau Numérique (SDD-000c)

> A'Space OS existe en deux exemplaires: le monde réel et le jumeau numérique. Toute modification passe d'abord par le jumeau.

Ce concept garantit la souveraineté: avant d'écrire du code, le design est dans le SDD. Avant le SDD, l'ADR. L'ordre est immuable.
