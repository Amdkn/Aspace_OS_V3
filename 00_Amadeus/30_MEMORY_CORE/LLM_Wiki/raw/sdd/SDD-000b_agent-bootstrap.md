### 6.2 Anti-Panic Patterns

| Framework | Problem | Solution |
|-----------|---------|----------|
| Claude Code | Approval prompts `[o/s/D]` | `bypassPermissions: true` in settings.json |
| Paperclip AI | Hard token budget | Explicit `budget` with soft/hard limits |
| OpenClaw | DM pairing blocks | `dmPolicy: "open"` + `allowFrom` list |
| Hermes | Shell freeze | `execute_code` with timeout |

## 10. Security

### 10.1 Protected Files

| Path | Access | Reason |
|------|--------|--------|
| `~/.ssh/*` | DENY | Private keys |
| `~/.hermes/vault/*` | DENY | A0 secrets |
| `/etc/shadow` | DENY | System passwords |
| `/etc/passwd` | DENY | User database |

### 10.2 API Key Management

- **Never store in code** — Use environment variables
- **LiteLLM config:** `os.environ/ANTHROPIC_API_KEY`
- **OpenRouter:** `os.environ/OPENROUTER_API_KEY`

## 12. Best Practices Summary

1. **Always set budget limits** — Never run without token/cost caps
2. **Use bypassPermissions sparingly** — Only for trusted internal agents
3. **Start services in order** — LiteLLM → Paperclip → Hermes → UI
4. **Check health before use** — Never assume services are running
5. **Document failures** — Log to Donna DLQ for triage
6. **Separate concerns** — OpenClaw for orchestration, Paperclip for control
7. **Escalate properly** — 3 attempts before DLQ
