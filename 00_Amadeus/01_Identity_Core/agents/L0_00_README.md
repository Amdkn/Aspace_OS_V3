# L0_00 — Protocole d’Activation A1/A2/A3

## 0) Pré‑requis
- A0 Local actif
- Sources de vérité chargées (README/SOUL/IDENTITY/USER/AGENTS)
- Contexte de mission défini (objectif + périmètre)

## 1) Déclencheur (A0)
A0 émet l’Intention :
> « Activer [A1] pour [mission] »

## 2) Activation A1 (Gatekeeper)
A1 valide :
- Alignement avec la mission
- Risques / interférences
- Priorité et limites

**Sortie A1** :
- ✅ Go / ❌ No‑Go
- Scope clair
- Règles de non‑parasitage

## 3) Activation A2 (Managers)
A2 découpe en domaines :
- L0 : Infra / Interface / Pipelines
- L1 : Meaning / Balance / Execution / Structure / Chaos / Liberation
- L2 : People / IT / Ops / Product / Growth / Finance / Legal

**Sortie A2** :
- Plan par domaine
- Resources nécessaires
- Dépendances

## 4) Activation A3 (Techniciens)
A3 exécute par micro‑tâches :
- Création de docs
- Indexation
- Checks de cohérence
- Rapport d’exécution

**Sortie A3** :
- Livrables + journal
- État (Done/Blocked)

## 5) Boucle de clôture
A0 reçoit :
- Résumé final
- Décision de retrait ou d’escalade
