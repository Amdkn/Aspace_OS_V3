## Décision : 5 Piliers ICP Solaris Canon

### Pilier 1 — Persona archétype "Technicien fondateur E-Myth"

**Source canon** : Takeout Gemini 2026-03 L3330-3340 (origin spec ICP Solaris).

**Définition** : Persona archétype AaaS Solaris = technicien qui a fondé une agence web/création/marketing/AI et qui se retrouve coincé entre production (faire) et direction (penser). Le profil type :

- **4 sub-types canoniques** (Takeout Gemini 2026-03 L3330-3340) :
  1. **Marketing Digital** — agence qui vend du social media / SEO / ads sans stack technique profond
  2. **Création** — agence créative / branding / identité visuelle sans delivery structuré
  3. **Web** — agence web / dev / e-commerce avec dépendance sous-traitance
  4. **AI Agencies** — agence qui revend expertise IA sans gouvernance ni conformité

- **Trauma fondateur** : passage de "je fais" à "j'ai une entreprise" — pas de système, pas de process, pas de pricing clair, dépendance au client-qui-paye-au-black.

- **Verbatim caractéristique** (Takeout Gemini 2026-05 L19229) : le technicien fondateur "décroche des projets qu'il ne peut pas facturer à leur juste valeur parce qu'il n'a pas de cadrage".

**D1 receipt** : origin spec verbatim `[2026-03:3330-3340]` — confidence **HIGH** (verbatim canon).

### Pilier 2 — Mantra doctrinal "L'illusion du sur-mesure"

**Source canon** : Takeout Gemini 2026-05 L19068, L19213, L25337 (verbatim ×3).

**Définition** : Le mantra "L'illusion du sur-mesure" est le **credo doctrinal** AaaS Solaris. Il signifie que **le client PME Solo Founder ne veut pas du sur-mesure** — il veut **un produit qui marche**, prêt à l'emploi, qui résout son problème sans qu'il ait à penser architecture.

**Verbatim occurrences** (×3 confirmation) :
- `[2026-05:19068]` "L'illusion du sur-mesure, c'est que chaque client pense avoir un cas unique alors que 80% des besoins sont standardisables"
- `[2026-05:19213]` "Le piège du technicien fondateur : croire que chaque projet mérite du sur-mesure. Faux. Le sur-mesure est l'ennemi de la scalabilité."
- `[2026-05:25337]` "Tarifer du sur-mesure quand on est solo founder, c'est se condamner à l'esclavage client. La solution AaaS : produit fini, config simple, prix fixe."

**Implication AaaS** : Solaris ne vend pas du consulting ou du dev sur-mesure. Solaris vend **un produit fini AaaS** (Agent-as-a-Service) avec config simple. Le sur-mesure est **anti-pattern**.

**D1 receipt** : verbatim ×3 `[2026-05:19068]`, `[2026-05:19213]`, `[2026-05:25337]` — confidence **HIGH** (canon doctrinal ×3).

### Pilier 3 — Marché haute entropie 136,1 Mds$ intégrateurs système

**Source canon** : Takeout Gemini 2026-05 L155599 (étude de marché 'The Builders' 2026).

**Définition** : Le marché cible ICP Solaris = **marché des intégrateurs système et agences web/AI 2026**, évalué à **136,1 Mds$** TAM (Takeout Gemini 2026-05 L155599 verbatim).

**4 sections étude de marché** (sister [`ADR-MARKET-STUDY-001`](./ADR-MARKET-STUDY-001_the-builders-2026.md)) :
1. **La Mutation** — marché en transition digitale accélérée post-2024
2. **L'Offre E-Myth** — agencies/web E-Myth = 80% des acteurs, pas de système
3. **Services Hybrides** — tendance 2026 = mix SaaS + services + AI agents
4. **Les 3 Modèles** — pricing canon = Solo Founder / Standard / Groupe (sister ADR-AAAS-PRICING-001)

**Caractérisation haute entropie** : marché fragmenté, no dominant player, faible standardisation, marges compressées sur le sur-mesure. **L'opportunité AaaS** = standardiser ce que le marché ne standardise pas (productisé services).

**D1 receipt** : TAM verbatim `[2026-05:155599]` = "136,1 Mds$ marché intégrateurs système" — confidence **HIGH** (verbatim canon).

### Pilier 4 — Architecture 3-ICP canon (Solaris/Nexus/Orbiter)

**Source canon** : Sister [`ADR-L2-AAAS-001`](./ADR-L2-AAAS-001_aaas-doctrine-3-variants-solarpunk.md) RATIFIED 2026-06-21 (3 Variants Solarpunk).

**Définition** : L'architecture AaaS Sisters Doctrine = **3 ICP distincts** alignés sur 3 positionnements marché :

| # | ICP | First | Sister Doctrine | Positionnement |
|---|---|---|---|---|
| 1 | **Solaris** | 🎨 Visual First / DAM | Solarpme Sister | Agences créatives / marketing / branding (besoin = identité visuelle + assets library) |
| 2 | **Nexus** | ⚖️ Data First / Conformité | Souverain Sister | Agences data / conformité / legaltech (besoin = pipeline données + AI-Act ready) |
| 3 | **Orbiter** | 🏗️ Mobile First / Terrain | Franchise Sister | Agences terrain / BTP / logistique (besoin = mobile-first + offline + capture terrain) |

**D1 receipt** : sister ADR-L2-AAAS-001 RATIFIED 2026-06-21 verbatim — confidence **HIGH** (canon sister).

### Pilier 5 — Killer Feature "Agentic Governance" (4 mécanismes)

**Source canon** : Takeout Gemini 2026-05 L156796-156844 (Killer Feature canon).

**Définition** : La **killer feature AaaS Sisters Doctrine** = **Agentic Governance** = gouvernance des agents IA conforme AI-Act (driver légal application totale **août 2026**).

**4 mécanismes canon** (verbatim `[2026-05:156796-156844]`) :

1. **Action Space Bounding** — définition stricte du périmètre d'action de chaque agent IA (no scope creep, no unauthorized actions)
2. **Sandboxing** — isolation des exécutions agents (no blast radius, no contamination inter-agents)
3. **HITL Dynamique** — Human-In-The-Loop adaptatif selon criticité (auto-approve pour low-risk, manual-approve pour high-risk)
4. **Traçabilité AI-Act** — log complet de chaque décision agent pour audit conformité (registre AI-Act article 12)

**Différenciateur marché** : aucun concurrent direct n'offre ces 4 mécanismes packagés en produit AaaS prêt-à-l'emploi (étude de marché 'The Builders' 2026, sœur `ADR-MARKET-STUDY-001`).

**D1 receipt** : verbatim `[2026-05:156796-156844]` — confidence **HIGH** (canon 4 mécanismes ×4 sous-verbatim).
