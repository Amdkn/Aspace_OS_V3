# Starter Specs

Fill-in templates that Claude Code reads as the handoff for your build. They pair with the Build-It Prompt Pack: most of these get written for you as you run the prompts in order, you just review and edit. Or fill the [brackets] by hand and point Claude Code at them.

## The flow

- **Part A (set up)** writes **SETUP.md** as you install the CLI, the toolchain, and Bedrock access from scratch.
- **Part B (plan)** writes **SYSTEM_SPEC.md**, **ARCHITECTURE_SPEC.md**, **COST_SPEC.md**, and **READINESS.md**.
- **Part C (draft)** uses **SECURITY_SPEC.md**, **AGENT_SPEC.md**, and **BUILD_PLAN.md** to scaffold and draft the build.

## The files

- **SETUP.md** - your from-zero setup facts (OS, CLI, account, region, bootstrap, model access). If a line is blank, you're not ready to build.
- **SYSTEM_SPEC.md** - what you're building: data, jurisdiction, surfaces, models, team, retention, budget, non-negotiables, tier.
- **ARCHITECTURE_SPEC.md** - the design: principles, the secure-substitute map, the data flow, and the ordered stack list.
- **SECURITY_SPEC.md** - the controls, mapped to the five categories.
- **COST_SPEC.md** - the two-number cost split plus the watch-list.
- **READINESS.md** - the five-category compliance readiness checklist, with the disclaimer.
- **AGENT_SPEC.md** - one block per agent.
- **BUILD_PLAN.md** - the phased plan, foundation first.

These are planning templates, not legal advice. Real compliance depends on your jurisdiction, industry, scope, and the auditor you choose.
