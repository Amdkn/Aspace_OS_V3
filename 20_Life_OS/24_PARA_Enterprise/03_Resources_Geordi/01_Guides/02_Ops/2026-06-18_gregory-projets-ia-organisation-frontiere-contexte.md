## 🧠 Concepts Clés à Haute Densité Sémantique

- **<Frontière de Contexte Stricte>** : Mémoire globale = repères généraux (style, langue). Projet IA = frontière stricte. Conversation = tâche spécifique. Fichier externe = indépendance. Stack 4 couches pour éviter le mélange entre projets professionnels et conversations personnelles. Chaque couche a un rôle précis.
- **<Espace Dédié Multi-Conversations>** : Projet IA = espace dédié (conversations multiples, fichiers, instructions, règles, exemples, fiches contexte) pour UN objectif. Toutes les conversations restent alignées sur même contexte sans réexpliquer. Économie cognitive majeure vs contexte global dispersé.
- **<Pattern 'Lis et Résume' — Vérification>** : Quand tu reviens dans un projet après un break, demande à l'IA de relire le contexte + résumer ce qu'elle a compris. Rend le contexte invisible → vérifiable. Si outdated, correction rapide (conversation) OU durable (fichier source markdown). Pattern de self-audit A'Space canonique.
- **<3 Limites Vitales des Projets IA>** : (1) Projet IA amplifie l'organisation, ne la remplace pas (fichiers obsolètes = garbage in), (2) Pas une poubelle à documents — qualité > quantité, (3) Contexte doit pouvoir voyager — copie externe markdown/Obsidian pour portabilité inter-outils.
- **<Méthode 5 Étapes — Initialisation>** : (1) Définir objectif, (2) ajouter fichiers essentiels (pas 20), (3) instructions projet pratiques, (4) conversation test pour résumer, (5) ajuster par itération. Pattern bootstrappable par tout A0 + A3 twins pour démarrer un projet sans overhead cognitif.
- **<Documentation Vivante via Conversations>** : Quand un bon échange avec l'IA clarifie un projet, demander une fiche markdown meta-contexte qui résume les infos importantes. Relire + corriger + ajouter au fichier projet. Chaque bonne conversation peut devenir une fiche contexte réutilisable. Anti-pattern = laisser les insights mourir dans l'historique.

## 🔬 Annexes Détaillées de l'Audit Sémantique & Technique

### Note Note A3-01 : Méthode 5 étapes appliquée à un projet A'Space concret

Exemple projet : Refonte OMK Tax Service dashboard. Étape 1 = définir 'dashboard financier pour kalybana communauté'. Étape 2 = ajouter UNIQUEMENT fichiers canoniques (CLAUDE.md + AGENTS.md + REBUILD_WORKFLOW.md + ADR-OMK-004 + ADR-OMK-001). Étape 3 = instructions 'toujours vérifier avec Tilly avant commit pricing'. Étape 4 = conversation test 'résume ce projet en 5 lignes'. Étape 5 = ajuster. 30 min setup = semaines de friction évitées.

### Note Note A3-02 : Documentation Vivante — Implémentation A'Space

Pattern Gregory appliqué : chaque skill créé via `/skill-creator` doit produire une fiche meta-contexte automatiquement. Cette fiche est commitée dans `wiki/hand_offs/skill_<name>/CONTEXT.md`. À chaque révision du skill, la fiche est mise à jour. Pattern = canon vivant, pas artefact mort.

### Note Note A3-03 : Anti-pattern 'Poubelle à Documents'

Erreur fréquente = ajouter tous les fichiers d'un dossier dans le projet IA. Le bon critère = chaque fichier doit être 'utile, fiable et à jour'. Fichier obsolète = bruit cognitif + risque d'hallucination. Auditer trimestriellement avec un A3 twin dédié (Boimler A3 triage).
