## 1. Domain Identity

### What This Domain OWNS

Relational Capital is the trust-based asset layer — named allies, verified track records, and reciprocity depth that funds all network activation capacity. It is the foundation upon which network activation (D02) operates.

### What This Domain Does NOT OWN

- Network activation protocols (that's D02)
- Experiential design (that's D03/D04)
- LD08 doctrine (that's D05-D07)
- Contribution architecture (that's D08)

### Contribution Type

Relational Capital contributes **trust assets** — the relational deposits that enable network activation when solarpunk objectives require coalition-building.

### Carnegie Primitives (Non-Negotiable)

| Primitive | Practice | Failure Mode |
|-----------|----------|--------------|
| **Name** | Use people's names correctly, consistently, with respect | Forgetting names, mispronouncing, using nicknames without permission |
| **Listening** | Full presence — mirror back, ask follow-up, don't interrupt | Pseudo-listening (waiting to talk), partial attention |
| **Smiling** | Warmth without falseness — "you exist and I am glad" | Performance smiles, transactional smiles, frozen smiles |

### Trust as Economic Accelerant

- High-trust networks compress transaction costs by 60-80%
- Trust enables asynchronous collaboration without verification overhead
- Trust compounds: each trustworthy interaction seeds the next
- **Operational rule**: Never make a trust claim you can't back. Never overpromise. Never assume trust — always build it.
