## 2. The Bar: 6 Standards Required

Any project or contribution claiming a solarpunk position must clear ALL of the following:

| Standard | Requirement | Verification |
|-----------|-------------|--------------|
| **Biomimetic foundation** | Explicit biological or natural system reference | Documented in project brief |
| **Circular output** | All outputs are inputs for other processes | Cascade diagram required |
| **Local resilience** | Solution works without global supply chains | Supply chain map required |
| **Knowledge contribution** | Network gains documented knowledge from this work | Contribution log required |
| **Relational reciprocity** | Give/Take ratio in network activation > 2:1 | Trust metric tracked quarterly |
| **Authentic fun** (if experiential) | Experience produces genuine play, not engagement metrics | User testimony required |

## 3. Anti-Extraction Checklist

Before launching any project under the Jerry Solarpunk umbrella:

| Question | If "Bad Answer" |
|----------|----------------|
| Does this contribute to the solarpunk commons, or only to revenue? | "Only revenue" = NOT Jerry Solarpunk |
| Is this designed to be locally resilient, or does it depend on global extraction? | Global dependency = NOT Jerry Solarpunk |
| Who benefits from the knowledge generated? Is it locked behind paywalls? | "Locked" = NOT Jerry Solarpunk |
| Is fun here authentic, or manufactured engagement? | "Manufactured" = NOT Jerry Solarpunk |
| Does this increase or decrease trust in the network? | "Decrease trust" = NOT Jerry Solarpunk |
| Can outputs return to a natural cycle or productive cascade? | "No cascade" = NOT Jerry Solarpunk |
| Is this solution adequate rather than maximally optimized? | "Maximum optimization at expense of accessibility" = NOT Jerry Solarpunk |

> If any answer is "only revenue," "locked," "manufactured," "decrease trust," or "maximum optimization at expense of accessibility," the project is NOT a Jerry Solarpunk contribution. It is commerce wearing solarpunk clothing.
