## 1. Domain Identity

### What This Domain IS

> **WARNING**: LD08 is A0 SOVEREIGN DOCTRINE. MUSE criteria carry A0 authority.

MUSE = Minimum Usable Solarpunk Experiment — the threshold test for solarpunk contribution. This domain operationalizes what qualifies as a MUSE-eligible contribution and what disqualifies it.

### What This Domain Does NOT OWN

Nothing. MUSE criteria are sovereign. Jerry stewards, A0 confirms.

### Contribution Type

MUSE-Eligible Criteria contributes **contribution threshold testing** — the qualifying and disqualifying rules that determine whether a contribution is MUSE-eligible.

---

## 2. The Separation

The L2 Business Pulse produces two types of output:

| Type | Definition |
|------|------------|
| **Commerce** | Revenue-generating activity that serves the operating entity |
| **MUSE-eligible contribution** | Public goods that serve the solarpunk commons |

> Commerce funds the operation. MUSE contributions define the operation's purpose.
> A business that only does commerce is not a solarpunk business. It is just a business.

---

## 3. The 7 Qualifying Rules (ALL must be present)

| # | Rule | Test |
|---|------|------|
| 1 | **Solarpunk thesis alignment** | Does this work advance regenerative rather than extractive systems? |
| 2 | **Public goods character** | Does this create value that others can use without exclusion? |
| 3 | **Relational capital growth** | Does this deepen trust or build new connections? |
| 4 | **Experiential capital deepening** | Does this produce genuine engagement, not just activity? |
| 5 | **Biomimetisme presence** | Is nature-inspired design in the approach, not just the output? |
| 6 | **Circular thinking** | Does this reduce waste or transform waste into input? |
| 7 | **Knowledge sharing intent** | Does this contribute to shared knowledge, not private advantage? |

---

## 4. The 5 Disqualifiers (ANY one is disqualifying)

| # | Disqualifier | Red Flag |
|---|--------------|---------|
| 1 | **Extraction primary** | Primary output is private profit with incidental public benefit |
| 2 | **Green veneer** | "Sustainable" claim without circular design or regenerative practice |
| 3 | **Relational exploitation** | Uses relationships for extraction rather than mutual benefit |
| 4 | **Play deprivation** | Demands contribution without any genuine play element |
| 5 | **Knowledge hoarding** | Creates knowledge but locks it rather than sharing it |

---

## 5. Public Benefit Bar

| Activity | Solar? | Why |
|----------|--------|-----|
| Public infrastructure that serves community | YES | Direct public benefit, zero exclusion |
| Open-source tool that reduces barrier to entry | YES | Knowledge sharing, shared benefit |
| Community garden that feeds residents | YES | Circular, local, closed-loop |
| Local energy cooperative | YES | Regenerative ownership model |
| Corporate sustainability report | NO | Green veneer, no structural change |
| "Eco-friendly" product that still creates waste | NO | Circular failure, green marketing |
| Extraction mining with carbon offset | NO | Externalized cost, not regenerative |
| Token "volunteer" program that replaces paid work | NO | Relational exploitation |
| Knowledge that stays in private hands | NO | Knowledge hoarding |
| Public narrative that shapes regenerative culture | YES | LD08 — cultural worldbuilding |
