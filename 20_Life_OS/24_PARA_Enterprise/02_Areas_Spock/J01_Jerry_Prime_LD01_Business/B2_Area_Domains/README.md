## Area-Level B2 Accountability

### Jerry's B2 Oversight Role

Jerry tracks:
1. **Domain Rock Status**: Each B2 manager's current Rock progress (weekly)
2. **Cross-Domain Dependencies**: When one domain's Rock blocks another
3. **Mode Alignment**: Whether current Solaris/Nexus/Orbiter mode fits domain mix
4. **Portfolio Impact**: How domain performance affects Area MRR, NPS, pipeline

### B2 Manager Activation Criteria

A B2 manager is "active" when:
- Domain folder structure confirmed (per domain README in B2_Area_Domains)
- KR scorecard live and updating weekly
- First Area-level Rock defined for this domain
- Cerritos routing confirmed for domain-related ideas

### Cross-Domain Integration

Domains are not siloed. Key cross-domain dependencies Jerry monitors:

- **Growth + Sales**: Pipeline coverage depends on brand investment and lead quality
- **Product + Ops**: Offer completion rate depends on Cerritos fidelity and system uptime
- **Finance + IT**: Runway decisions depend on tech debt ratio and infrastructure investment
- **People + Ops**: Team retention depends on system health and process clarity
