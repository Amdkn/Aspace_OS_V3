# Built To Sell Scorecard

## Purpose

A Summer project can be exciting and still not sellable. This scorecard checks whether it is becoming an asset instead of a personalized workload.

## Score Fields

Use 0-3 for each field.

| Field | 0 | 1 | 2 | 3 |
|---|---|---|---|---|
| Niche clarity | vague | broad | specific ICP | painful niche + urgency |
| Repeatable offer | custom each time | partial template | packaged offer | productized buying path |
| Delivery independence | founder-only | assisted | SOP-backed | operator-ready |
| Revenue logic | unknown | rough price | margin model | validated willingness to pay |
| Lead source | none | manual | repeatable channel | measured acquisition loop |
| Handoff proof | none | notes | B2/B3 packets | external operator can execute |

## Gate

If total score is below 12, the project remains PRODUCT_ONLY_PROTOTYPE or SERVICE_EXPERIMENT. If 12-15, it can enter SOB_INCUBATION. If 16-18, it can enter OPERATING_BUSINESS_CANDIDATE.

## Evidence Required

Each score must link to a local proof path, Baserow/Plane record, customer artifact, SOP, commit, call note, invoice, dashboard, or explicit hypothesis marker.
