# B2 Domain 08 — Cognition/Mastery

## Domain Identity

### What It Protects

Cognition/Mastery is the **deep work and cognitive load management domain** — it protects:
- Deep work capacity (90min+ uninterrupted cognitive blocks)
- Working memory efficiency (chunking, not overload)
- Cognitive load ceiling (ability to handle complexity without collapse)
- Emotional regulation during learning (anxiety management)
- Learning velocity under stress (desirable difficulty)
- Long-term memory integration (schema consolidation)

### Why It Matters

```
Deep work 90min → equivalent to 3–4h of fragmented cognitive work
Working memory limit → >4 chunks = cognitive overload = zero learning occurring
Cognitive load >8/10 → decision quality equivalent to blood alcohol 0.08%
Anxiety >7 during learning → hippocampus-dependent encoding disabled
Desirable difficulty → optimal struggle zone produces 2–3× better retention
Long-term integration → weekly review of learned concepts maintains 80% retention
```

## Jerry's Stewardship Protocol

### Deep Work Protocol

```
Daily Minimum (GREEN):
  □ 90min deep work (single task, no multitasking)
  □ 1× 90min block OR 2× 45min blocks (no splitting further)
  □ No meetings, no email, no Slack during deep work
  □ Phone on airplane mode

Optimal Structure:
  □ Morning (best cognitive capacity): deep work on hardest problem
  □ Mid-day: meetings and coordination (low cognitive demand)
  □ Afternoon: second deep work block OR focused learning
  □ Evening: no cognitively demanding work (sleep preparation)

Environment Standards:
  □ Single screen (no second monitor with notifications)
  □ Browser closed except the single task application
  □ Noise-canceling headphones (or silence)
  □ Temperature: cool (20–22°C — warmth degrades alertness)
  □ Light: natural or bright artificial (dimness = drowsiness)
```

### Cognitive Load Management

```
Pre-Session Check (mandatory):
  □ Rate cognitive load 1–10 before starting
  □ Rate emotional state 1–10 before starting
  □ If cognitive load >7 OR anxiety >7 → abort session, do low-demand task only

During Session:
  □ Chunk enforcement: if problem requires >4 chunks simultaneously, pause
  □ Desirable difficulty: task should feel effortful but solvable
  □ If cognitive load spikes to 8+ → 10min break, not power through
  □ If anxiety spikes to 8+ → abort session entirely (counterproductive to continue)

Post-Session:
  □ Rate perceived difficulty 1–10 (calibrate for next session)
  □ Log session: task, duration, completion, load score, notes
  □ If session completion <60% → investigate root cause before next attempt
```

## Escalation Rules

```
ORANGE Trigger: Deep work <90min for 3+ days OR cognitive load >7 for 3+ sessions
  → ORANGE ADVISORY: cognitive bandwidth is constrained
  → Reduce: cognitive business load (meetings, decisions, negotiations)
  → Implement: pre-session load check (abort if >7)
  → Review: sleep debt (sleep ORANGE → cognition ORANGE within 48h)

ORANGE Trigger: Session completion <60% for 2+ sessions
  → ORANGE ADVISORY: tasks being attempted are too complex
  → Break: divide complex task into 25min sub-tasks
  → Reduce: cognitive load ceiling to 5/10 maximum per sub-task
  → If cannot reduce load: defer task entirely until ORANGE clears

RED Trigger: Cognitive load >9 for 2+ consecutive days OR anxiety >8 during learning
  → HARD FREEZE: learning acquisition prohibited
  → Beth notification if correlated with LD03 ORANGE
  → Identify root cause: sleep debt? emotional overload? overcommitment?
  → Recovery: low-demand tasks only, no new acquisition
  → Full clearance required before resuming normal cognitive work
```
