# Summer's Verse - Picard Project Template

Summer lives in Picard's Projects because Summer is finite execution: a client, product, OS variant, sprint, or business build with a clear outcome.

## Doctrine

- A Summer's Verse is a bounded active Project.
- It inherits the same fractal as Jerry: B1 Direction -> B2 DC domain managers -> B3 Marvel execution squads.
- It must link to a 12WY Rock, a GTD next-action queue, and a DEAL graduation/automation path.
- It cannot be opened directly from raw Jerry strategy; Cerritos must clarify the work first.

## Structure

| Folder | Role |
|---|---|
| `B1_Summer_Direction` | Local CEO/project direction, scope, constraints, acceptance |
| `B2_Business_Domains` | Eight DC/Marvel domain folders used by every business project |

## Instantiation Rule

Current Picard projects `00` to `05` are not modified in this pass. This folder is the root template and handoff surface. A future pass can instantiate the template into each active project once A0 selects the first project to formalize.
