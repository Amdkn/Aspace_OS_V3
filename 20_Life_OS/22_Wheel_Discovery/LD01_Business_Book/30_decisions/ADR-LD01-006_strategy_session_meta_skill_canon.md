## Decision

### 2.1 Lock design canonique (10 décisions verrouillées)

**D1 — SKILL canon RATIFIED** : `~/.claude/skills/strategy-session-meta/` est la source canonique des Scripts Phase 1 + runbook 7 étapes + 7 règles dures. Append-only sister canon propagé à `LD01/10_methodology/00_strategy_session_meta_skill.md`.

**D3 — Phase 2 EXECUTE** : runbook 7 étapes (parse → D1-locate → execute outillage existant → 0-result = anomaly D6 → fix + LESSONS → receipts D5 → WEEK_CONTRACT). Sisters : ADR-LD01-005 §3.2 (cost reality soft guardrail) + ADR-LD01-004 §C5 (kill-switches).

**D4 — 6 principes v2 sister convergence** : Decision Block (P1) + Wager (P2) + D5 receipts (P3) + Sister chain (P4) + Calibration interview (P5) + Fable 5 self-audit (P6) appliquées aux reports CC via W5+W6+W7 sister propagation.

**D5 — Append-only sessions/** : metrics + questions + HTML datés (D1 verification commit chaque lundi). Pas de hard-delete. Sister : CARDIA-TDD §A Additif.

**D6 — LESSONS.md canonique** : append-only registry des leçons seedées (5 leçons actuelles). Chaque drift observé pendant Phase 2 EXECUTE = 1 append. Sister : CARDIA-TDD §A Antifragile (system gains from stress).

**D7 — WEEK_CONTRACT.md vivant** : 3 moves + statut (fait / gated A0 / à toi) + suivi semaine courante. Lundi suivant → Phase 1 avec nouvelles métriques.

**D8 — 7 règles dures CC** : (1) question sans fait mesuré = interdite, (2) format rapport exact, (3) zone faible = focus semaine, (4) blunt ≥ 7 → blunt_line, (5) runbook 7 étapes, (6) 0-result = anomaly D6, (7) append-only LESSONS.md.

**D9 — Activation production gated 3 sessions consécutives** : W5 (2026-07-07) + W6 (2026-07-14) + W7 (2026-07-21). Si les 3 sessions fonctionnent sans escalade A0 → activation production continue (Pattern ADR-LD01-004 §C3.3). Si l'une échoue → amendement ADR ou HOLD.

## Consequences

### Positives

- **Loop complet canonisé** : mesure → confrontation → correction → canonisation = procédurable chaque lundi.
- **2 instances sisters convergence** : W3 v2 (qualitative) + W4 v1 (opérationnelle) = même template Fable #4, deux implémentations convergentes.
- **Validation canon d'Eero** : CC a construit ce skill en autonomie (Niveau 3 autonomy demonstration) — la doctrine CARDIA-TDD tient structurellement.
- **Append-only DATA** : sessions/, contracts/, LESSONS.md = historisation complète D1-verifiable.
- **Cross-harness agnostic** : `collect_metrics.py` et `render_session.py` en Python stdlib = n'importe quel harness (CC/MC/HA/Shadow L1) peut les invoquer.

### Tradeoffs

- **Phase 1.5 v2 application** est manuelle (Mavis doit appender les 6 principes au questions.json CC chaque lundi). Mitigation : automation gated W7 (script Python sister).
- **Runbook 7 étapes Phase 2 EXECUTE** demande A0 HITL explicite pour le 0-result (étape 4) — A0 doit répondre à la question "anomalie ou comportement?" avant que CC continue.
- **3-sessions gate** = patience (3 semaines) avant production continue — vs CARDIA-TDD §A Antifragile qui demande speed. Tradeoff assumé.
