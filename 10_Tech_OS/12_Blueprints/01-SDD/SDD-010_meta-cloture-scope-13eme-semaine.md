## 3. La Doctrine de la 13ème Semaine

### 3.1 Origine arithmétique

Le 12 Week Year (12WY) de Brian Moran est calé sur 12 semaines = 1 trimestre intensif. Or l'année calendaire fait **52 semaines** = **4 × 12 + 4 semaines de "pause"**.

→ **Entre chaque cycle 12WY se glissent ~1 semaine de transition** : la **13ème semaine**.

C'est :
- ✅ La revue méta-stratégique du cycle terminé
- ✅ Le repos cognitif imposé (anti-Burn)
- ✅ La promotion Muse V0 → V1 si DoD atteinte
- ✅ La rédaction du cycle suivant (Quarter Intent + 12 Rocks + Warp Core W1-W12)
- ✅ Le moment où Amadeus est **autorisé à ne rien produire** sans culpabilité

### 3.2 Mapping H1 — Calendrier A'Space 2026

| Cycle 12WY | Période | Type | Notes |
|---|---|---|---|
| **Cycle 1** (Genesis) | 2026-01 → 2026-03-22 | Production | Stabilisation L0, Constitution SDD-000→007 |
| **13ème Sem #1** | 2026-03-23 → 2026-03-29 | Pause Meta | Revue Q1 + planification Q2 |
| **Cycle 2** (Hospital Planet) | 2026-03-30 → 2026-06-21 | Production | **Cycle ACTUEL** — Conscience L1 + monitoring biométrique |
| **13ème Sem #2** | 2026-06-22 → 2026-06-28 | Pause Meta | Revue Q2 + planification Q3 |
| **Cycle 3** (SOB Factory) | 2026-06-29 → 2026-09-20 | Production | Action L2 + premier cashflow |
| **13ème Sem #3** | 2026-09-21 → 2026-09-27 | Pause Meta | Revue Q3 + planification Q4 |
| **Cycle 4** (Sovereign Loop) | 2026-09-28 → 2026-12-20 | Production | Graduation MUSE + 7 jours autonomie |
| **13ème Sem #4** | 2026-12-21 → 2026-12-27 | Pause Meta | Bilan H1 complet + planification H2 |

### 3.3 La 13ème Semaine de cette session

Aujourd'hui, **2026-05-13** est **techniquement dans le Cycle 2** (Hospital Planet, W7/12). Mais SDD-010 acte une **13ème Semaine d'exception** :

> *"Quand le serveur reste à 100% CPU pendant 2+ jours et que la souveraineté absolue se révèle être du sable mouvant, on déclenche une 13ème semaine d'exception — peu importe la position dans le calendrier 12WY."*

→ **La 13ème semaine n'est pas seulement calendaire — elle est aussi déclenchable par crise méta.**

### 3.4 Critères de déclenchement d'une 13ème Semaine d'Exception

1. **Crise infrastructurelle** : VPS down, OOM, throttle Hostinger > 24h
2. **Crise cognitive** : Amadeus identifie un piège conceptuel majeur (ex: "souveraineté absolue")
3. **Crise de production** : aucune valeur générée en L2 pendant 7 jours consécutifs (Beth Veto)
4. **Crise architecturale** : un SDD est identifié comme mal classé (cf. reclassement OpenHarness)
5. **Fin de cycle MUSE** : graduation V0 → V1 d'un composant

→ Toute 13ème Semaine d'Exception **doit produire un SDD Meta** ou un ADR-META documentant la leçon.

## 4. La Leçon Centrale — Fin de la Souveraineté Absolue

### 4.1 Le piège conceptuel V0

Pendant les Cycles 1-2 (Q1 + début Q2 2026), Amadeus a poursuivi un idéal :

> **"Tout doit être self-hosted, open source, hors-cloud, immédiatement souverain."**

Conséquences observables :
- VPS Hostinger surchargé jusqu'à 100% CPU pendant 32+ heures (cf. ADR-RICK-001 historique)
- Paperclip AI en crashloop 117 restarts
- 4 mois passés à 80% en L0 (bedrock) au lieu de L2 (valeur)
- Aucun cashflow généré
- Sentiment de "construire un empire sur du sable mouvant"

### 4.2 La correction Beth Veto + SDD-010

> **"V0 = sable mouvant accepté. V1 = antifragile + souverain + durable."**

| Dimension | V0 (avant SDD-010) | V1 (cible post-13ème semaine) |
|---|---|---|
| Sovereignty | Absolue (tout self-hosted dès J1) | **Graduelle** (Shadow Cloud → MUSE → Self-Hosted) |
| Frugalité | All-Claude ($300+/mois) | **Stack chinoise** MiniMax 2.7 + GLM 4.7 ($<30/mois) |
| Bedrock | Fragile (1 VPS = 1 point de défaillance) | **Antifragile** (Symphony + multi-tenant + cloud fallback) |
| Production | Bloquée tant que L0 imparfait | **Continue** en Shadow L2 pendant que L0 mûrit |
| Identité | Architecte solitaire | **Pair Amadeus + Claude Architect + Stack agentique** |

### 4.3 Ce qui n'a PAS changé

La souveraineté **reste un objectif** — elle est juste **déplacée dans le temps** :
- Self-hosted reste la cible MUSE de chaque outil
- Open source reste la préférence (Plane, Baserow, Affine OSS)
- Souveraineté du LLM via stack chinoise (MiniMax = pas US-dependent)
- Souveraineté des données via Supabase self-hosted (déjà en place)

**Ce qui est mort** : l'urgence absolutiste qui poussait à coder en L0 au lieu de produire en L2.

## 6. Le Veto SDD — 90 jours sans nouveau SDD

### 6.1 Énoncé

> **À partir du 2026-05-13, aucun nouveau SDD (au-delà de SDD-010) ne peut être créé pendant 90 jours, soit jusqu'au 2026-08-11.**

### 6.2 Justification

- 13 SDDs = scope COMPLET pour V0 (cf. §2 inventaire)
- Ajouter un SDD avant la 13ème Semaine #2 (juin 2026) = retomber dans le perfectionnisme de la souveraineté absolue
- Le travail productif des 90 jours doit aller dans **PRD / ADR / DDD / TDD** (les couches d'exécution sous-SDD)

### 6.3 Exception : un seul SDD-META autorisé

Si une **13ème Semaine d'Exception** est déclenchée (cf. §3.4 critères), un nouveau SDD-META peut être écrit pour documenter la leçon. Convention de numérotation : `SDD-010b`, `SDD-010c`, etc. (pas `SDD-011` qui briserait le scope).
